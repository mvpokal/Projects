TSA Complaints Analysis Project
Project Overview

This project analyzes TSA (Transportation Security Administration) complaint data to identify trends, patterns, and common issues reported by travelers. The goal is to better understand passenger concerns and evaluate areas where airport security processes may need improvement.
Objectives

    Clean and prepare TSA complaint data for analysis
    Identify common categories of complaints
    Analyze trends over time and across locations
    Explore patterns in passenger experiences
    Visualize key insights using data visualization techniques

Tools & Technologies

    Python
    Pandas
    NumPy
    Matplotlib / Seaborn
    Jupyter Notebook

Project Structure

    TSA Complaints Code.ipynb → Main analysis notebook
    TSA Compliants Paper.pdf → Written summary of findings (note: filename typo in original file)

Key Steps

    Loaded TSA complaints dataset into Python
    Cleaned and standardized data for analysis
    Categorized complaints by type and frequency
    Analyzed trends over time and by category
    Created visualizations to highlight key patterns
    Summarized findings in a written report

Results & Insights

    Certain complaint categories appeared more frequently than others
    Trends showed variation in complaints over time
    Some locations and categories had consistently higher complaint volumes
    Data visualization helped clearly identify problem areas in TSA processes

Limitations

    Data may not represent all passenger experiences
    Some entries may be incomplete or inconsistently recorded
    Analysis is descriptive and does not imply causation

Key Takeaways

This project demonstrates how data analysis can be used to evaluate customer experience in public service systems. It highlights how structured analysis and visualization can help identify operational improvement areas.
