# SparkPi with PySpark Project

## Project Overview
This project demonstrates distributed computing using Apache Spark with PySpark by estimating the value of π (Pi) using a Monte Carlo simulation approach. It showcases Spark’s ability to perform parallel processing across distributed data partitions.

## Objectives
- Understand the basics of Apache Spark and PySpark  
- Implement a Monte Carlo simulation to estimate π  
- Execute distributed computations using Spark RDDs  
- Analyze terminal outputs from Spark job execution  
- Document results and workflow behavior  

## Tools & Technologies
- Apache Spark  
- PySpark  
- Python  
- Jupyter / Terminal  
- Distributed Computing Concepts  
- Word Documentation

## Key Steps
1. Initialized Spark session using PySpark  
2. Defined Monte Carlo simulation logic for estimating π  
3. Distributed computation across Spark RDD partitions  
4. Executed Spark job and collected terminal outputs  
5. Aggregated results to compute final π approximation  
6. Documented workflow and interpretation of results  

## Results & Insights
- Spark successfully distributed computation across multiple partitions  
- Monte Carlo simulation produced a close approximation of π  
- Terminal output confirmed successful execution of Spark job stages  
- Demonstrated Spark’s efficiency in handling parallel workloads  

## Limitations
- Simulation accuracy depends on number of iterations used  
- Environment is likely local or single-node Spark setup  
- No cluster-level performance benchmarking included  
- Focus is on demonstration rather than optimization  

## Key Takeaways
This project demonstrates how Apache Spark can be used with PySpark to perform distributed computations efficiently. It highlights the power of parallel processing through a simple Monte Carlo simulation to estimate mathematical constants.
